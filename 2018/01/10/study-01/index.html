干货，语法

reactDOM.render

我想reactDOM应该是react-dom的全局api，通过调用自身render方法将dom元素渲染到浏览器中。
用法很简单，值得注意的是，render方法html内容不能加引号，且只能有一个根标签。

JSX 允许直接在模板插入 JavaScript变量。如果这个变量是一个数组，则会展开这个数组的所有成员。

React.createClass

此大法，大言不惭，在我看来应该是react语法的核心大法
贴上代码示例吧，否词下次看不懂了。

首先，像下面这样创建一个组件类

var HelloMessage = React.createClass({
  propTypes:{
      name:React.PropTypes.string.isRequired
  },
  render: function() {
    return 
    <div>
    <h1>Hello {this.props.name}</h1>
    <p>some text</p>
    </div>;
  }
});

创建好了，向下面这样用，这里的name是自定义属性，在组件类中可以通过props得到自定义的属性

ReactDOM.render(
    <HelloMessage name="is it a shit?">,
    document.getElementById('example')
)

this.props.children

该属性在组件类内部作为变量来使用，children并不指向自定义的属性，我将其理解为组件的内部属性，用来代表组件的所有子节点

propTypes

类型属性过滤性，指定自定义属性的类型，不符合类型将报错,getDefaultProps方法可以用来设置组件属性的默认值，用法如下：

var MyTitle = React.createClass({
  getDefaultProps : function () {
    return {
      title : 'Hello World'
    };
  },
  render: function() {
     return <h1> {this.props.title} </h1>;
   }
});
ReactDOM.render(
  <MyTitle />,
  document.body
);

上面，虽然在render组件时未给出title的值，但其拥有默认值，页面显示‘Hello World’。

React中的DOM节点

很久之前就听说过react虚拟DOM的说法，react的DOM diff算法将真实DOM和虚拟DOM之间建立起连接，使得在虚拟DOM上发生的变动都能迅速的在真实DOM上反映，据说可以极大的提高网页的性能，但了解不深，此时我还不信。

在组件类内部的refs属性，指向所有定义了ref属性的节点，分别根据定义的名字保存，可以通过refs[name]得到真实的DOM节点。

var MyComponent = React.createClass({
  handleClick: function() {
    this.refs.myTextInput.focus();
  },
  render: function() {
    return (
      <div>
        <input type="text" ref="myTextInput" />
        <input type="button" value="Focus the text input" onClick={this.handleClick} />
      </div>
    );
  }
});

ReactDOM.render(
  <MyComponent />,
  document.getElementById('example')
);

this.state

在react的组件有这样一种概念，组件状态，当组件与用户进行交互时，难免需要保留组件特有的状态信息，普通JS中我们可以通过一个全局变量去实现，但既然是框架肯定会提供统一的方法来定义组件的状态，组件类中通过getInitialState方法实现初始状态的定义，同时通过this.state属性来得到状态值，this.setState方法来改变对应属性值。

var LikeButton = React.createClass({
  getInitialState: function() {
    return {liked: false};
  },
  handleClick: function(event) {
    this.setState({liked: !this.state.liked});
  },
  render: function() {
    var text = this.state.liked ? 'like' : 'haven\'t liked';
    return (
      <p onClick={this.handleClick}>
        You {text} this. Click to toggle.
      </p>
    );
  }
});
ReactDOM.render(
  <LikeButton />,
  document.getElementById('example')
);

React组件的生命周期

简单的说三个状态：

  • Mounting: 已插入真实DOM
  • Updating: 正在被重新渲染
  • Unmounting: 已移除真实DOM

每一个状态提供两种处理函数，will函数在进入状态之前调用，did在之后调用

  • componentWillMount()
  • componentDidMount()
  • componentWillUpdate(object nextProps, object nextState)
  • componentDidUpdate(object prevProps, object prevState)
  • componentWillUnmount()

两种特殊的处理函数(暂时还不清楚什么情形该用)

  • componentWillReceiveProps(oject,nextProps):已加载组件收到新的参数时调用
  • shouldComponentUpdate(object nextProps, object nextState)：组件判断是否重新渲染时调用

  • 获取数据

最后贴上一大段代码，实在不想写了。
这是React结合promise的用法

var RepoList = React.createClass({
  getInitialState: function() {
    return { loading: true, error: null, data: null};
  },

  componentDidMount() {
    this.props.promise.then(
      value => this.setState({loading: false, data: value}),
      error => this.setState({loading: false, error: error}));
  },

  render: function() {
    if (this.state.loading) {
      return <span>Loading...</span>;
    }
    else if (this.state.error !== null) {
      return <span>Error: {this.state.error.message}</span>;
    }
    else {
      var repos = this.state.data.items;
      var repoList = repos.map(function (repo) {
        return (
          <li>
            <a href={repo.html_url}>{repo.name}</a> ({repo.stargazers_count} stars) <br/> {repo.description}
          </li>
        );
      });
      return (
        <main>
          <h1>Most Popular JavaScript Projects in Github</h1>
          <ol>{repoList}</ol>
        </main>
      );
    }
  }
});
ReactDOM.render(
  <RepoList
    promise={$.getJSON('https://api.github.com/search/repositories?q=javascript&sort=stars')}
  />,
  document.body
);
